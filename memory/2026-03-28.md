- Added MongoDB Lambda container scaffolding:
  - `scripts/mongo_versions.txt` – list of MongoDB versions (default 8.3.0-rc3) for easy parameterisation.
  - `docker/Dockerfile.mongo` – Ubuntu‑based Dockerfile that downloads the specified MongoDB binary, exposes `mongod` and `mongo` executables, and copies the Lambda handler.
  - `lambda/mongo_handler.py` – Python Lambda handler that starts `mongod` with optional flags, runs a JavaScript payload via the `mongo` shell, captures the output, shuts down `mongod`, and returns the result.
  - `scripts/run_mongo_container.sh` – helper script to build the container (optionally with a chosen version), create a tiny example JavaScript file, run the image locally and print the Lambda‑style JSON response.
  - Added usage notes for building locally, testing with the example script, pushing the image to ECR, and creating the Lambda function.
- Provided step‑by‑step guidelines for testing the MongoDB‑Lambda container locally (run_mongo_container.sh, custom JS script, direct Python handler).
- Fixed build error by changing default `MONGODB_VERSION` to `6.0.13` in `sandbase/docker/Dockerfile.mongo`, a known working arm64 release.
